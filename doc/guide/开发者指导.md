## 本地数据库可视化

执行 `npm run infra:up` 后，会额外启动一个本地 `pgAdmin`：

- URL: `http://127.0.0.1:5050`
- Email: `admin@example.com`
- Password: `adminadmin`

在 `pgAdmin` 中添加当前项目 PostgreSQL 连接时，推荐填写：

- Host name/address: `postgres`
- Port: `5432`
- Maintenance database: `iot_claw`
- Username: `postgres`
- Password: `postgres`

这样可以直接查看 `devices`、`device_state`、`device_alerts`、`device_commands` 等业务表，用于本地开发和调试。

## 关键开发原则

- 优先通过服务层和桥接层扩展，不在 HTTP 层直接拼业务 SQL
- 新增配置时同步更新 `src/config.ts`、`.env.example` 和相关文档
- 保持严格类型，避免引入 `any`
- 高风险路径优先考虑策略、审批和审计
- 先保护 MQTT -> 存储 -> HTTP API 主链路，再扩展 bridge 和 agent 能力

## Bridge 使用

当前支持两种运行时：

- `AGENT_RUNTIME=openclaw`
- `AGENT_RUNTIME=disabled`

`0.0.5` 开始新增 `openclaw` 原生插件集成骨架，建议优先通过 `plugins/openclaw-iot-claw/` 对接 `iot-claw` HTTP API，而不是继续扩展外部 skill 或 `nanoclaw` 兼容路径。

联调 `openclaw` 时建议检查：

- `OPENCLAW_GATEWAY_URL`
- `OPENCLAW_GATEWAY_TOKEN`
- `OPENCLAW_CLIENT_ID`
- `data/openclaw/identity/device.json`
