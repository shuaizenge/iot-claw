# 用户指南

## iot-claw 是做什么的

`iot-claw` 的目标，不只是把设备接进来，而是让设备能够被 Agent 系统理解和调度。

对普通用户来说，理想使用方式应该是：通过 npm 安装、完成最少配置、直接启动服务，而不是先理解仓库内部实现。

它适合这样的场景：

- 你有一批通过 MQTT 接入的设备，需要统一查看状态和告警
- 你希望给设备下发命令，并保留审批与审计记录
- 你希望让 `openclaw` 参与设备调度或通过插件调用设备能力
- 你希望把硬件能力变成一个更稳定的控制面，而不是散落在脚本里

## 你通常会怎么使用它

最常见的路径是：

1. 让设备通过 MQTT 接入
2. 在控制面查看设备状态、命令、任务和 bridge 事件
3. 在需要时发起命令，并走审批或审计链路
4. 把告警、任务或设备能力桥接给上层 Agent

## 普通用户关注什么

普通用户通常只需要关心：

- 怎么安装
- 怎么配置环境变量
- 怎么启动服务
- 怎么接入设备
- 怎么调用接口或接入上层 Agent

## 常见场景

### 设备接入与状态查看

- 采集设备状态和事件
- 写入遥测到 InfluxDB
- 在 HTTP control plane 查询设备状态

### 命令下发与审批

- 对设备下发控制命令
- 对高风险命令走审批流程
- 保留执行结果和审计记录

### Agent 协同

- 将告警或任务桥接到 `openclaw`
- 让 Agent 通过控制面查询设备和下发动作
- 把设备变成 Agent 工作流中的末梢执行器

# 插件安装

安装步骤
1. 安装本地插件
openclaw plugins install -l /home/zengshuai/project/cursor/iot-claw/plugins/openclaw-iot-claw
2. 启用插件
openclaw plugins enable iot-claw
3. 配置 iot-claw 地址
openclaw config set plugins.entries.iot-claw.config.baseUrl "http://127.0.0.1:8080"
4. 如果你给 iot-claw 配了 API Token，再配置同一个 token
openclaw config set plugins.entries.iot-claw.config.apiToken "你的-token"
5. 可选：配置默认租户和站点
openclaw config set plugins.entries.iot-claw.config.defaultTenant "default"
openclaw config set plugins.entries.iot-claw.config.defaultSite "default"
6. 重启 gateway
openclaw gateway restart
验证
pnpm openclaw plugins list
pnpm openclaw plugins inspect iot-claw
补充说明
- -l 是 link 模式，适合你现在本地开发这个插件。
- 之后你改 plugins/openclaw-iot-claw/ 里的代码，重启 openclaw gateway 就能重新加载。
- 如果你的 OpenClaw 配了 plugins.allow，记得把 iot-claw 加进去。
- 如果 iot-claw 没设置 apiToken，插件调用 /api/plugin/* 时可以先不配 token。
